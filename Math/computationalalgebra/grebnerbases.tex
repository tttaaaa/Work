\section{Introduction}
\label{sec:Introduction}
グレブナー基底について調べ，その理解をまとめる．
このノートでは，グレブナー基底が数学において，どのように使われるのか，また，どのような部分が問題なのかを
探し，まとめていくものである，そのため章ごとの関連ははないことに注意して欲しい．
グレブナー基底を中心にした私の現時点の興味は２点ある．
1つは定理の自動証明だ．アルゴリズム的に解けるか否かを判定させ，それを用いて問題を解くというものである．
コンピュータが発展した今だからこそできるようになった分野であること，またアルゴリズム的に解ける，ひいては有限と無限の
間を与える問題に感じるためである．
もう1点はグレブナー基底自体のアルゴリズムの拡張である．
グレブナー基底は多項式にのみ考えられているが，それを拡張したい．
例えば，一般のネーター環に対してどの程度グレブナー基底のような方法ができるのか．
そこまでは難しくとも，べき級数環や，多項式の係数が整数やp進数の場合の定式化に興味がある．
また，付値論的な方法でグレブナー基底らしいものを作れないかという興味ある．
完備離散付値環，例えば $\mathbb{Z}_p$に対し，元の計算を楽にする方法がないか．
例えばその元の付値が簡単に計算できるアルゴリズムがないかを考えたい．
これらが見えてくると計算機における理解が非常に進むように感じる．

\subsection{多項式の割り算}
\label{sub:多項式の割り算}
１変数多項式環$K[X]$には次数が定義される．
この次数を用いて，割り算ができる．
\begin{rem}
 割り算はどう解釈するべきだろうか．例えば最大公約数のように考えると，割り算はだし，
 これはユークリッド整域に思えるのか．ユークリッド整域だと割り算が計算できるアルゴリズムがあるということ．元に着目してるんだよなぁ．完全な余談です．
\end{rem}

多変数の場合にも同様に割り算を定義したい．
\begin{yodan}
 なぜ？割り算を定義したい？．一種の同一視．自然な拡張ができるか．
 特に割り算がうまく定義できれば，イデアルの元かの判別ができるため，
 一つの元に対して計算が可能になる．
\end{yodan}

多項式の場合は次数による順序の概念が自然には定義できない．
そのため都合のいい順序を考える必要がある．
1変数の場合と同様な多項式にしたい．

1変数の類似で直感的に思い浮かぶ性質は２つ．
\begin{itemize}
  \item 1変数に制限した時に多項式の次数で比較できること
  \item 任意の二元に対して比較ができること．
\end{itemize}

グレブナー基底では以下の条件を満たす単項式順序を入れる．

\begin{dfn}
 単項式順序とは，
 \begin{itemize}
   \item $\alpha > \beta$ ならば,$ \alpha + \gamma > \beta + \gamma$となる.
   \item 整列可能順序になっている．つまり最小限が存在する．
 \end{itemize}
\end{dfn}
\begin{rem}
 整列可能順序ってこれでいいんだっけ？
\end{rem}

例として辞書式順序等がある．

\begin{thm}
 多変数多項式で割り算するアルゴリズムが存在する．すなわち，
\begin{equation*}
    g = a_1f_1 + \cdots + a_nf_n +r
\end{equation*}
とかける．ここで $a_if_i$の次数は$g$の次数以下となる．
\end{thm}
\begin{rem}
 $r$の次数が小さいとは限らない． 
\end{rem}
